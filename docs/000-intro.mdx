# Start here

Gurukul is a self-evolving documentary of LLM research. Every chapter on the
left sidebar — except this one — is generated by a two-model loop running
against your own Databricks Foundation Model endpoints. The journey grows
one topic at a time, the way you'd actually learn a fast-moving field.

## The protocol

A **Teacher** model outlines the chapter and critiques the draft. A
**Student** model writes the draft, then revises after the critique. The
final JSON is rendered through a strict MDX template, so the build can
never break on a stray brace from the model. You decide which protocol
to use per topic:

- `outline-write-critique` (default) — the richest, slowest, most worth-it.
- `socratic` — every aspect opens with a question.
- `debate` — pro vs con drafts get synthesised into a final chapter.

## What every topic page gives you

Each generated chapter has a TL;DR, a few "key aspects", small gists
(code or math), an experiment you can run, **open problems** you can
star, **references** you must verify, a **pushback box** for your
disagreements, a **research seed** capture, and a **confidence tracker**.

The pushback box and the seeds are the two most important widgets — they
are how this site stops being a study journal and starts becoming the
seed of a paper you write yourself.

## What it is not

- Not a tutor for visitors of this public site. Content is a snapshot of
  *your* journey; new content is generated on your machine and committed.
- Not a NeurIPS-paper generator. It produces an outline. The experiments
  are still on you.
- Not grounded. The FM has no web access; it speaks from training data.
  That's why the References widget defaults to *unverified* and asks you
  to mark each one. Don't cite anything you haven't verified.

## How to start

```bash
cp .env.example .env
# fill in DATABRICKS_HOST, DATABRICKS_TOKEN, endpoint names
npm install
npm run journey      # interactive: generate your first topic
npm start            # local preview
```

When you're ready, browse to **Journey Dashboard** (top nav) for an
aggregated view, and **Proposal Outline** when you're ready to draft.
